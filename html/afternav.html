Overview

Stellar is open source. The network is comprised by people like you running stellard, the stellar server.

There are 3 ways to issue commands to stellard.

  • Command line- This is the usual method when you are running stellard locally.
  • RPC- stellard can be configured to listen for http requests on an arbitrary port. You can then make normal PUT requests with the command and parameters sent as a JSON string in the data.
  • Websocket- stellard can also be configured to create a websocket. It is useful if you want to subscribe to events from stellard. For example if you want to know when a particular account gets a transaction. All websocket commands can take an optional id integer parameter. This id will be reflected in stellard's response.
Stellar Foundation runs two instances of stellard you can use if you don't want to run your own.

amount

Many of the API commands accept an argument of amount type. An amount is either an amount of stellars (STR) or an amount of some other currency.

An amount of stellars is always represented in microstellars (the smallest divisible stellar unit).

example


 55 stellars:
   "amount" : "55000000"

An amount that represents a currency credit is a triplet of the following parameters:

  • value: the amount of credit.
  • currency: the 3 letter currency code, USD, BTC, EUR etc.
  • issuer: the Stellar account that issued the credit.

example


 34.5 BTC:
 "amount" : {
    "value" : "34.5"
    "currency" : "BTC",
    "issuer" : "ganVp9o5emfzpwrG5QVUXqMv8AgLcdvySb",
 }
